# מדריך מסך האפליקציה — "The Void / Solar System Survival"

> **הערה:** מסמך זה מתאר את **החלק הויזואלי של המשחק בלבד** — ללא לוח הצ'אט של ה-AI וללא חלון ה-Debug של ה-Engine State.

---

## מה רואים ברגע שהאפליקציה עולה?

כאשר פותחים את הדפדפן ומנווטים ל-`http://localhost:5173`:

### רקע — סצנת החלל התלת-ממדית
- המסך **כולו** מכוסה בסצנת WebGL של חלל אפל.
- כ-**800 כוכבים** (נקודות לבנות) פזורים על כדור סביב המצלמה.
- המצלמה ממוקמת מרחוק למעלה ולפנים: `(0, 2000, 3000)` — נקודת מבט של ציפור מעל המערכת השמשית.
- אם מנוע ה-Rust **לא רץ עדיין**, מופיע בריבוע מרכזי אפור כהה:

```
[ Activity Icon ]
CONNECTING TO ENGINE...    ← אם WebSocket מתחבר
      –– או ––
   THE VOID IS EMPTY       ← אם החיבור נכשל
```

## צעד אחרי צעד — מה רואים

### שלב 1 — טעינת הדפדפן
- הרקע שחור לרגע קצר.
- Three.js / WebGL מאתחל את הסצנה.
- **800 כוכבים** מופיעים על הרקע.
- WebSocket מנסה להתחבר לשני שרתים:
  - **Rust Engine** — `ws://127.0.0.1:8081/ws`
  - **Python Director** — `ws://127.0.0.1:8000/api/v1/dream-stream`

---

### שלב 2 — מצב "Void" (לפני חיבור)
- באמצע המסך: ריבוע כהה עם הודעה `THE VOID IS EMPTY` או `CONNECTING TO ENGINE...`
- פרטי HUD כבר גלויים אך ריקים:
  - **פינה שמאלית-עליונה** (אחרי לוח השמאל): פס HULL INTEGRITY ירוק מלא — **100**
  - **פינה ימנית-עליונה** (שורת כפתורים): סמל 🗑, כפתור Rachel: ON, 💾, 🔊, ועיגול צהוב מהבהב + הכיתוב `Disconnected. Reconnecting...`

---

### שלב 3 — חיבור מנוע ה-Rust
- עיגול הסטטוס בפינה ימנית-עליונה משנה צבע מ**צהוב** ל**ירוק** ומפסיק להבהב.
- הכיתוב משתנה ל: `Connected to Director` (כשגם ה-Python מחובר).
- ה-WebGL Canvas ממשיך להציג כוכבים אבל עדיין ריק מישויות.

---

### שלב 4 — הופעת ישויות ראשונות (World Spawn)
הרגע שבו מנוע ה-Rust מספון את העולם — כל ישות מגיעה עם **אנימציית לידה**:
- היא מופיעה **קטנה וסופר-בהירה** (scale 0.1, brightness ×5) ומתרחבת לגודל מלא תוך **1.2 שניות**.
- ישויות ראשוניות אופייניות:
  - **שמש** — כדור צהוב זוהר במרכז `(0, 0, 0)` עם אור חזק.
  - **כוכבי לכת** — כדורים כחלחלים-ירקרקים בגלי מסלול הליוצנטריים.
  - **אסטרואידים** — גופים אפורים נעים בסינוסואיד.
  - **אוניית השחקן** — מת"ל לבן בצורת משולש / GLTF mesh.

---

### שלב 5 — אוניית השחקן על המסך
- האוניה מופיעה ב-`(0, 0, 0)` כברירת מחדל.
- המצלמה **עוקבת אחריה** באופן דינמי: offset `(0, 400/zoom, 600/zoom)` עם הצמדה רכה (lerp 4%).
- ברדאר בפינה ימנית-תחתונה: **משולש לבן קטן** מייצג את מיקום השחקן.

### שלב 7 — שליטה בתנועה
המשתמש לוחץ על מקשי המקלדת:

| מקש | פעולה |
|---|---|
| `W` / `↑` | תנועה קדימה |
| `S` / `↓` | תנועה אחורה |
| `A` / `←` | תנועה שמאלה |
| `D` / `→` | תנועה ימינה |
| `Space` | ירייה (לייזר) |
| **גלגלת עכבר** | זום מצלמה (0.2 – 5.0) |

- המקשים נשלחים בזמן-אמת כ-JSON למנוע ה-Rust.
- המצלמה מתעדכנת בהדרגה אחרי האונייה.

### שלב 8 — אירועי משחק על המסך

#### התנגשות עם אסטרואיד / אויב:
- **רעידת מסך** — 0.2 שניות של אנימציית shake.
- **הבזק אדום** — צל אדום סביב הקצוות של המסך (vignette effect) — 0.55 שניות.
- **פס הבריאות** יורד:
  - ≥ 60% → **ירוק**
  - 30–60% → **צהוב**
  - ≤ 30% → **אדום** + הכיתוב `⚠ CRITICAL` מהבהב מתחת לפס
- **צליל התנגשות** — תדר + גבר לפי מהירות ומרחק.
- **אנימציית ירי לייזר**: חלקיקים כחולים-לבנים יוצאים מהאונייה.

#### הרג אויב:
- הישות "מתמוטטת" — **אנימציית implosion** 1 שנייה (מתכווצת לאפס + מאירה).
- מספר ה-**KILLS** עולה ב-1.

### שלב 9 — Game Over

כאשר הבריאות מגיעה ל-0:

```
         S I G N A L   L O S T
         Rebooting Neural Link...
```

- הכיתוב האדום מהבהב במרכז המסך (אנימציית pulse).
- המשחק קופא — אין יותר קלט.

### שלב 10 — שמירת העולם

לחיצה על כפתור ה-💾 בפינה ימנית-עליונה:

1. שולח `POST http://127.0.0.1:8080/save` לשרת ה-Rust.
2. מופיע **Toast** מעל הכפתורים למשך 3 שניות:

```
💾  Universe Saved to Disk
```

> ה-Rust שומר את World State הנוכחי לקובץ `world_snap.json` בשורש הפרויקט.
